HTTP

HTTP概念

HTTP，Hypertext Transfer Protocol，超文本传输协议。其基于TCP（Transmission Control Protocol）协议，TCP是一种面向连接的，基于字节流的传输层通信协议。HTTP属于应用层协议。

HTTP报文

请求报文

  • 请求行：包含请求方法（如GET、POST）、请求的URL、HTTP协议版本
  • 请求头：包含请求的元数据信息，以键值对的形式表示
  • 空行：用于分隔请求头和请求体
  • 请求体：用于传输请求的具体数据内容，如表单数据、JSON数据等

响应报文

  • 状态行，包含HTTP协议版本、状态码、状态信息
  • 响应头，包含响应的元数据信息，以键值对的形式表示
  • 空行，用于分隔响应头和响应体
  • 响应体，用于传输相应的具体数据内容，如HTML、JSON数据等

常见状态码
  • 200 OK：请求成功，服务器成功处理了请求。
  • 201 Created：请求成功，并在服务器上创建了新的资源。
  • 204 No Content：请求成功，但响应报文中没有实体内容返回。
  • 400 Bad Request：请求有语法错误或参数错误，服务器无法理解。
  • 401 Unauthorized：请求需要身份验证，客户端未提供有效的身份凭证。
  • 403 Forbidden：服务器拒绝执行请求，没有访问权限。
  • 404 Not Found：请求的资源不存在。
  • 500 Internal Server Error：服务器内部错误，无法完成请求。
  • 502 Bad Gateway：服务器作为网关或代理，从上游服务器收到无效响应。
  • 503 Service Unavailable：服务器当前无法处理请求，通常是因为过载或停机维护。

HTTP/0.9

该协议仅仅限于文字，图片等信息是无法显示和处理。

HTTP/1.0

该协议可处理HTML外其他类型内容，但每次请求需要建立新的TCP连接。

HTTP/1.1

引入了持久连接，即在一个TCP连接上可以发送多个请求和响应，减少了连接建立和关闭的开销。

另外支持管线化方式（pipelining)处理，可以同时并行发送多个请求。

HTTP/2.0

该协议采用二进制协议传输数据，引入了多路复用特性，允许并发地发送多个请求和响应。

另外支持头部压缩、服务器推送等功能，提高传输效率和性能。

HTTP/3.0

该协议基于QUIC（基于UDP，不需要握手和挥手），实现了0RTT。

HTTPS

是在HTTP协议基础上添加了TLS/SSL加密层，通过使用证书和加密算法来保护通信的安全性。
